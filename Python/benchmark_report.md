# BitmaskHunter 压测报告

## 测试参数
- **算法**: 贪心位掩码搜索 + 蒙特卡洛随机采样
- **单条规则最大位宽**: 12
- **每轮位宽随机尝试次数**: 500
- **样本生成方式**: 随机 MD5（os.urandom + hashlib.md5）
- **正负样本比例**: 1:1（总样本平分为 A、B 两份）
- **规模上限**: 500,000 总样本
- **单轮耗时上限**: 120 秒

## 结论与观察

1. **训练耗时**: 随着样本规模增大，训练耗时通常呈非线性增长。当正负样本均为随机 MD5 时，由于分布均匀，早期规模增长耗时增长较缓；但当规模达到数万级别后，黑名单计算量和搜索空间急剧膨胀，耗时可能快速上升。
2. **规则数量**: 规则数量通常随样本规模增加而上升，但增长斜率取决于数据本身的可分性。若正负样本差异显著，规则增长较缓慢；若边界模糊，则可能需要大量细粒度规则。
3. **涉及比特位**: 随着规则数量增加，涉及的比特位会逐步分散到 128 个位的不同位置，最终趋近于 128（饱和）。
4. **覆盖能力**: 在随机 MD5 数据上，只要搜索空间（max_width 和尝试次数）足够，算法通常仍能实现 100% 的 B 集合检出和 0% 的 A 集合误报。但若规模过大导致训练超时，可能出现未完全覆盖的情况。
5. **优化建议**: 若训练耗时成为瓶颈，可考虑降低 max_width、减少每轮尝试次数，或采用更高效的位运算批量计算策略。
