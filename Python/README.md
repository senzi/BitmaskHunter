# BitmaskHunter (Python 版)

Python 参考实现，提供从样本生成、掩码训练、效果验证到性能压测的完整工具链。

---

## 作用

在实际业务中，我们常常需要判断一个 MD5 哈希是否属于某个特定集合（例如黑名单、特征库等）。直接存储全部哈希值会占用大量内存，而 BitmaskHunter 通过提取少量关键比特位构成规则，能够以极低的存储成本和计算开销实现高精度的集合成员判定。

具体来说，BitmaskHunter 可以：

- **区分两组样本**：给定一组"正样本"（必须命中）和一组"负样本"（必须排除），自动学习出位掩码规则。
- **压缩存储**：规则仅保存少数比特位的索引和期望模式值，远比完整哈希列表节省空间。
- **快速匹配**：匹配时只需提取固定比特位并进行整数比较，速度极快。

---

## 原理

MD5 哈希本质是一个 128 位整数。BitmaskHunter 的核心思想是：

1. **位模式（Bit Pattern）**：从 128 位中选取少量比特位（例如 5~12 位），这些位上的 0/1 组合构成一个模式值。
2. **黑名单过滤**：对于选定的比特位，先计算负样本集合中所有哈希的模式值，形成"黑名单"。
3. **正样本匹配**：如果某个正样本的模式值不在黑名单中，则该 `(bits, val)` 组合可以作为一条安全规则——它能覆盖所有具有该模式值的正样本，同时保证不会误伤负样本。
4. **贪心覆盖**：每次找到一条规则后，从正样本中移除所有被覆盖的样本，然后在剩余的样本上继续搜索，直到全部覆盖或无法继续。
5. **随机搜索**：由于 128 位中选取特定位的组合空间巨大，算法采用蒙特卡洛随机搜索，在可接受的位宽范围内快速寻找有效规则。

## 文件说明

| 文件 | 作用 |
|------|------|
| `00_gen_samples.py` | 生成随机 MD5 样本，输出 `A.txt`（负样本）和 `B.txt`（正样本） |
| `01_train_mask.py` | 贪心搜索训练位掩码规则，输出 `rules.mask`，并打印统计报表 |
| `02_benchmark.py` | **压测脚本**：动态增量测试不同样本规模下的训练耗时与规则规模，生成 `benchmark_report.md` |
| `03_verify.py` | 加载 `rules.mask`，对 A/B 集合进行误报/检出率验证 |

## 快速开始

### 1. 生成样本

```bash
cd Python
python 00_gen_samples.py
```

运行后会生成 `A.txt` 和 `B.txt`，分别包含 100 条随机 MD5 哈希（默认共 200 条，各一半）。

- **A.txt**：负样本，规则**不应**命中这些哈希。
- **B.txt**：正样本，规则**需要**命中这些哈希。

> 你也可以使用自己的业务数据替换这两个文件，每行一个十六进制 MD5 字符串。

### 2. 训练掩码规则

```bash
python 01_train_mask.py
```

程序会执行贪心搜索，在控制台输出每条规则的位宽和覆盖情况，训练结束后：

- 规则保存到 `rules.mask`。
- 同时输出 **Mask 生成报表**，包括：
  - 训练总耗时
  - 规则总数量
  - 涉及多少个不重复的比特位（共 128 位）
  - 各规则的位宽分布

**规则文件格式示例：**

```
0,5,10:3
12,33,44,55:7
```

每行格式为 `bit1,bit2,...,bitN:value`，表示选取这些比特位，当它们的组合值等于 `value` 时视为命中。

### 3. 验证规则

```bash
python 03_verify.py
```

脚本会加载 `rules.mask`，并对 `A.txt` 和 `B.txt` 进行验证，输出：

- 规则总数量
- 集合 A 的误报数量和比例
- 集合 B 的检出数量和比例
- 是否完美区分两组样本

## 压测（Benchmark）

`02_benchmark.py` 用于评估算法在不同样本规模下的表现。它会：

- 从 200 条样本起步，根据上一轮训练耗时**动态决定下一轮规模**（5x / 2x / 1.5x）。
- 记录每轮的样本规模、训练耗时、规则数量、涉及比特位、位宽分布、检出率、误报率。
- 最终输出 `benchmark_report.md` 报表。

```bash
python 02_benchmark.py
```

**动态增量策略：**

| 本轮耗时 | 下一轮规模倍数 |
|---------|--------------|
| < 5 秒  | 5 倍 |
| < 30 秒 | 2 倍 |
| < 120 秒 | 1.5 倍 |
| >= 120 秒 | 停止 |

## 参数调整

在 `01_train_mask.py` 中，你可以调整以下参数：

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `max_width` | 单条规则最多使用的比特位数 | `12` |
| `500`（搜索次数） | 每个位宽下的随机尝试次数 | `500` |

- **增大 `max_width`**：可以提升区分能力，可能用更少的规则覆盖全部样本，但匹配时计算量略增。
- **增大随机尝试次数**：提高找到最优规则的概率，但训练时间会更长。

---

## 注意事项

1. **样本差异**：如果集合 A 和集合 B 的哈希分布过于相似（例如有大量重合），算法可能无法找到完美区分的规则，此时会输出 `[WARNING] 无法完全覆盖所有样本`。可以尝试增大 `max_width` 或增加随机搜索次数。
2. **随机性**：由于采用随机搜索，每次训练得到的规则可能不同，但只要能完美区分，规则都是有效的。
3. **通用性**：本算法不仅限于 MD5，任何固定长度的二进制指纹（如 SHA-256、SimHash 等）均可适用，只需调整位数范围即可。
